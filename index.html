Give your AI agents memory they can verify.

Agents act on stored memory. But that memory can be edited, overwritten, or silently corrupted. verifiable-memory-mcp gives an agent an integrity layer: it checks memory before acting, stops if it was changed outside the approved flow, and exports portable evidence of what happened.

How it works

Verify → Stop → Export → Verify again

Every memory entry is append-only, timestamped, and cryptographically linked to the one before it — a hash chain. Any later change to a past entry breaks the link and becomes detectable.

01

Verify

Before acting, the agent re-checks the integrity of its operational memory.

02

Stop

If memory was modified outside the approved flow, the agent halts instead of acting on it.

03

Export ECO

It writes a portable evidence receipt: what it relied on and what the integrity check found.

04

Verify ECO

Anyone can re-check that receipt independently — no backend, no trust in our screen.

Demo

Don't trust me — verify it yourself.

The verifier is a single static page. Load an exported bundle and it recomputes the whole chain in your own browser with WebCrypto. Green means intact. Change one character and it turns red. Try it on your own phone.

Evidence

A portable package, not a screenshot.

The main evidence artifact is an .eco package: the exported bundle, an integrity anchor, agent decision metadata, and a human-readable report. The verifier checks it entirely in the browser.
