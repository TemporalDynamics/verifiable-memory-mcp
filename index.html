Give your AI agents memory they can verify.

Agents act on stored memory. But that memory can be edited, overwritten, or silently corrupted. verifiable-memory-mcp gives an agent an integrity layer: it checks memory before acting, stops if it was changed outside the approved flow, and exports portable evidence of what happened.

How it works

Verify → Stop → Export → Verify again

Every memory entry is append-only, timestamped, and cryptographically linked to the one before it — a hash chain. Any later change to a past entry breaks the link and becomes detectable.

01

Verify

Before acting, the agent re-checks the integrity of its operational memory.

02

Stop

If memory was modified outside the approved flow, the agent halts instead of acting on it.

03

Export ECO

It writes a portable evidence receipt: what it relied on and what the integrity check found.

04

Verify ECO

Anyone can re-check that receipt independently — no backend, no trust in our screen.

Evidence

The ECO is the proof.

An ECO is a portable evidence receipt. It captures what happened, what the agent relied on, and whether the artifact can be independently verified. The stop is the behavior; the ECO is the proof; the verifier is the credibility.

artifact005_STOP_BY_INTEGRITY.eco
integrityTAMPERED
lifecycleOWNER_REVIEW
agent decisionSTOP_BY_INTEGRITY
reasonmemory modified outside approved flow
VALID TAMPERED UNKNOWN
What this does and doesn't promise. The system does not guarantee the content is correct — it guarantees that no one altered it without that being detectable. Integrity is not correctness. It is tamper-evident, not tamper-proof: every change is detectable. The public anchor is what stops someone from rewriting the whole local store.

Where it fits

Anywhere agents act on memory they need to prove.

  • Recruiting agents — trace why a candidate was recommended, with verifiable reasons.
  • Internal operations — prove why an autonomous workflow stopped or continued.
  • Compliance review — preserve evidence behind agent-assisted decisions.
  • Memory governance — detect unauthorized changes before the agent acts.
